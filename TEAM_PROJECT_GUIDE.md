# TEAM_PROJECT_GUIDE

## 1. 프로젝트 개요

이 프로젝트는 **주가 예측 모델 성능 비교**를 목표로 한다.  
단순히 하나의 모델만 만드는 것이 아니라, 아래 4개의 모델을 같은 데이터 기준으로 비교한다.

1. **고전적 모델**
2. **AI 기반 모델**
3. **심리 지수만 이용한 모델**
4. **AI 기반 + 심리 지수 결합 최종 모델**

---

## 2. 데이터 사용 기준

현재 팀에서 합의한 기본 데이터 수집 기준은 다음과 같다.

- **데이터 소스:** `yfinance`
- **기간:** **최근 5년**
- **기본 가격 데이터 컬럼:**
  - `Date`
  - `Open`
  - `High`
  - `Low`
  - `Close`
  - `Volume`

이 기본 OHLCV 데이터로 먼저 **고전적 모델**과 **AI 기반 모델**을 구축하고,  
이후 뉴스, 댓글, 커뮤니티 글 등에서 추출한 감성 점수나 투자 심리 지수를 추가해  
**심리 지수 모델**과 **결합 모델**을 확장하는 구조로 진행한다.

---

## 3. 모델별 역할 정리

### 1) 고전적 모델

과거 가격 데이터와 시계열 패턴을 활용하는 전통적 통계 기반 모델

**확정 모델:**
- **ARIMA (AutoRegressive Integrated Moving Average)**
  - 주기성, 추세성 등을 고려하여 가장 기본이 되는 기준(Baseline) 모델로 확정.

### 2) AI 기반 모델

과거 가격 데이터와 파생 변수들을 입력으로 활용하는 머신러닝 기반 모델

예시

- Random Forest
- XGBoost
- LSTM

### 3) 심리 지수만 이용한 모델

가격 데이터 대신, 투자 심리 지수만 입력으로 사용하는 모델

예시 입력

- 뉴스 감성 점수
- 댓글 감성 점수
- 커뮤니티 감성 점수
- 거래량 기반 심리 지수

### 4) AI 기반 + 심리 지수 결합 최종 모델

가격 데이터, 기술적 지표, 심리 지수를 함께 결합하는 최종 모델

---

## 4. 현재 프로젝트 진행 방식

현재는 **프로젝트 구조와 기본 코드 뼈대가 먼저 만들어진 상태**다.  
즉, 모든 파일이 완성본이라기보다 **역할별로 나뉜 기본 틀**이 들어가 있는 상태라고 보면 된다.

지금 단계에서 중요한 것은 다음 두 가지다.

1. **각 파일이 무슨 역할인지 팀원 전체가 이해하기**
2. **어떤 모델이 어느 파일에서 구현될지 정리하기**

## 6. 폴더별 역할 설명

### `configs/`

프로젝트 설정값을 모아두는 폴더

- `settings.py`
  - 파일 경로
  - 기본 파라미터
  - 공통 상수
  - 추후 기업명, 예측 대상, 모델 설정값 등을 정리할 수 있음

### `data/`

데이터 저장 폴더

- `raw/`
  - yfinance에서 처음 받아온 원본 주가 데이터 저장
  - 예: 삼성전자, NVIDIA, Lockheed Martin 등 5년치 주가 데이터

- `processed/`
  - 전처리 완료 후 저장되는 데이터
  - 결측치 제거
  - 파생 변수 생성
  - 학습용 데이터셋 저장

- `sentiment/`
  - 뉴스, 댓글, 커뮤니티 데이터
  - 감성 점수
  - 투자 심리 지수 관련 데이터 저장

### `notebooks/`

실험용 Jupyter Notebook 저장 폴더

- 초기 탐색
- ADF 검정 확인
- 시각화
- 모델 비교 실험

### `results/`

실험 결과 저장 폴더

- `metrics/`
  - 모델별 성능 지표
  - RMSE, MAE, 정확도 등 저장

- `predictions/`
  - 실제값과 예측값 결과 저장

- `figures/`
  - 그래프, 시각화 결과 저장

## 7. `src/` 파일별 역할 설명

### `__init__.py`

`src` 폴더를 하나의 패키지처럼 사용하기 위한 파일

---

### `data_loader.py`

데이터를 불러오는 파일

주요 역할

- CSV 불러오기
- yfinance로 받아온 데이터 로드
- 날짜 정렬
- 필요한 컬럼만 선택

---

### `preprocess.py`

전처리 관련 파일

주요 역할

- 로그수익률 생성
- 단순수익률 생성
- 타깃 변수 생성
- 결측치 제거

---

### `feature_engineering.py`

가격 데이터에서 추가 변수 만드는 파일

주요 역할

- 이동평균선 생성
- 변동성 생성
- 거래량 변화율
- 가격 범위
- 기타 기술적 지표

---

### `stationarity.py`

정상성 검정 관련 파일

주요 역할

- ADF 검정
- 정상성 확인
- 시계열 모델에 넣기 전 데이터 상태 점검

이 파일은 특히 **고전적 시계열 모델**에서 중요하다.

---

### `modeling.py`

기존 ARIMA 실험용 또는 전반적인 시계열 모델 테스트용 파일

주요 역할

- ARIMA 적합
- 기본 예측
- 탐색용 모델링

현재 구조상 `classical_model.py`와 일부 역할이 겹칠 수 있으므로,  
추후에는 **실험용 파일**로 남기거나 **고전적 모델 전용 공통 함수 파일**로 정리할 수 있다.

---

### `classical_model.py`

고전적 모델 전용 파일

주요 역할

- ARIMA/ARMA
- 전통 통계 기반 모델 학습
- 고전적 모델 예측 함수 분리

---

### `ai_model.py`

AI 기반 모델 전용 파일

주요 역할

- Random Forest
- XGBoost
- 기타 머신러닝/딥러닝 모델

---

### `sentiment_index.py`

심리 지수 생성 파일

주요 역할

- 뉴스 감성 점수 통합
- 댓글 감성 점수 통합
- 커뮤니티 감성 점수 통합
- 최종 투자 심리 지수 생성

---

### `sentiment_only_model.py`

심리 지수만을 입력으로 사용하는 모델 파일

주요 역할

- 심리 지수만 기반으로 예측
- 가격 데이터 없이 심리 신호만으로 예측

---

### `hybrid_model.py`

가격 데이터 + 심리 지수 결합 모델 파일

주요 역할

- 기술적 지표 + 심리 지수 결합
- 최종 모델 구현

---

### `split.py`

시계열 데이터 분할 파일

주요 역할

- train / validation / test 분리
- 시간 순서 유지 분할

주의

- 시계열 데이터는 랜덤 셔플하면 안 됨

---

### `diagnostics.py`

모형 진단 파일

주요 역할

- 잔차 진단
- Ljung-Box 검정
- 모델 적합 상태 확인

주로 **고전적 모델 진단**에서 사용됨

---

### `evaluate.py`

평가 파일

주요 역할

- RMSE
- MAE
- MAPE
- 방향성 정확도
- F1-score 등 계산

모든 모델 평가를 같은 기준으로 맞추는 역할을 함

---

### `train.py`

전체 학습 흐름을 묶는 파일

주요 역할

- 모델별 학습 순서 정리
- 공통 학습 파이프라인 작성
- 여러 모델을 한 번에 실행하도록 연결

---

### `utils.py`

공통 보조 함수 파일

주요 역할

- 출력 형식 통일
- 공통 유틸 함수
- 반복적으로 쓰는 작은 함수 정리

## 8. 현재 코드 상태에 대한 설명

현재 각 파일에는 **기본 뼈대 코드가 이미 들어가 있는 상태**다.  
즉, 완전히 빈 파일이 아니라 **역할별 초안 코드가 들어간 구조**라고 보면 된다.

따라서 지금 팀원들이 이해해야 하는 것은 다음과 같다.

- 이미 기본 구조는 잡혀 있다
- 앞으로 해야 할 일은 각 파일에 들어 있는 초안을 프로젝트 방향에 맞게 수정하는 것이다
- 특히 `modeling.py`와 `classical_model.py`는 역할이 겹칠 수 있으므로 정리가 필요하다
- 최종적으로는 **4개 모델을 명확히 분리해 비교할 수 있도록** 코드 구조를 통일해야 한다

## 9. 추천 작업 순서

### 1단계: 가격 데이터 기반 기본 모델 완성

먼저 yfinance 5년 데이터를 사용해 아래 두 모델을 먼저 구축한다.

- **고전적 모델 (ARIMA 확정)**: `data_loader.py`를 통해 데이터를 수집하고, `stationarity.py`를 통해 정상성 확인 후, `classical_model.py` / `modeling.py`에서 최적 파라미터(p,d,q)를 찾아 학습.
- AI 기반 모델: Random Forest, XGBoost 등 베이스라인 머신러닝 학습.

### 2단계: 심리 데이터 수집 및 심리 지수 생성

그다음 아래를 진행한다.

- 뉴스 수집
- 댓글 수집
- 커뮤니티 글 수집
- 감성 분석
- 심리 지수 생성

### 3단계: 심리 기반 모델 구축

이후 아래 두 모델을 만든다.

- 심리 지수만 이용한 모델
- AI + 심리 지수 결합 모델

### 4단계: 최종 비교

마지막으로 네 모델을 같은 기준으로 비교한다.

- RMSE
- MAE
- 방향성 정확도
- 결과 해석

---

## 10. 팀원들이 기억해야 할 핵심 정리

1. 이 프로젝트는 ARIMA 하나만 만드는 프로젝트가 아니다.
2. 총 4개의 모델을 비교하는 프로젝트다.
3. 기본 가격 데이터는 yfinance 최근 5년 기준이다.
4. 현재 코드는 역할별 기본 뼈대가 이미 들어가 있다.
5. 각 파일은 완성본이 아니라, 앞으로 수정하고 확장해 나갈 작업 기반이다.
6. 모든 모델은 같은 기간, 같은 평가 기준으로 비교해야 한다.
